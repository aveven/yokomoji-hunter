## コア機能（要件）

### ユーザーの操作フロー
1. 横文字に遭遇する（例：「LLM」「CVR」など）
2. アプリを開いて用語を入力する
3. 以下の情報が自動表示される（KBデータから）：
   - 重要度ランク（S/A/B/C）
   - 実務インパクト（0〜100点、バーアニメーション）
   - 遭遇率（高/中/低）
   - 中学生でもわかる解説
   - 使用例
   - 「覚えるとどうなる？」補助メッセージ（例：「これを理解すると、今後の調べ直しコストを大きく減らせる可能性があります」）
   - 獲得XP
4. 用語をコレクションに登録する → 喜びの演出（アニメーション等）
5. 後日クイズ（4択）で復習 → 正解でさらに報酬

### ゲーミフィケーション要素
- XP（経験値）・レベルシステム
- ストリーク（連続学習日数）
- レベルアップ演出
- 用語を「見つけた」時のワクワク感の演出

## 用語データ（KB）の構造

```js
// 用語データ.js
window.TERM_DATA = {
  "LLM": {
    fullName: "Large Language Model",
    reading: "えるえるえむ",
    cat: "AI",           // AI / マーケティング / ビジネス / IT / その他
    rank: "S",           // S / A / B / C
    impact: 95,          // 0〜100
    freq: "高",          // 高 / 中 / 低
    explanation: "中学生でもわかる解説文",
    example: "使用例",
    value: "覚えるとどうなる？（補助メッセージ）"
  },
  // ...
};
```

**重要：**
- `index.html`（アプリ本体）と`用語データ.js`（データ）は**別ファイルに分離する**
- HTMLから `<script src="用語データ.js">` で読み込む方式
- ローカルファイル（`file://`プロトコル）でも動作するよう、JSONではなくJSファイルにする
